# 银行股估值模型系统 (Bank Valuation Model System)

## 项目概述

这是一个**完整的、由数据驱动的、模块化的银行股估值分析系统**，专门用于对**A股上市银行**进行全方位的估值评估。该系统集成了四大经典金融估值模型，能够自动识别出被低估和高估的银行股，为投资决策提供科学支持。

### 核心特性

✅ **四大估值模型完全实现**
- DDM（股息贴现模型）
- PB-ROE模型（市净率模型）

- 相对估值法（PE/PB倍数法）

## 四大估值模型详解

### 1. DDM（股息贴现模型）- 最基础的绝对估值

**核心公式:**
```
P0 = D1 / (r - g)
转化为PB形式: PB = (ROE × d) / (r - g)
```

其中：
- P0：当前股价（或内在价值）
- D1：下一年预期每股股息
- r：股权成本（CAPM，约10%）
- g：永续增长率（约4%）
- d：派息率

**优点:** 直观，适合高分红银行
**缺点:** 对增长率假设敏感

### 2. PB-ROE模型 - 银行股最核心的模型

**单阶段简化公式:**
```
PB = (ROE - g) / (r - g)
```

**两阶段模型:**
```
ln(PB0) = (ROE - r) × T + ln(PBT)
```

其中：
- ROE：净资产收益率
- T：高增长期年限（通常5年）
- PBT：终端期PB（通常趋近1）

**这是最常用的银行估值模型！**

### 3. RIV（剩余收益模型）- 最严谨的会计视角

**核心公式:**
```
V0 = BV0 + Σ(ROEt - r) × BVt-1 / (1+r)^t
```

其中：
- BV0：当前每股净资产
- (ROEt - r)：超额回报（剩余收益）
- 核心思想：价值 = 账面净资产 + 未来超额收益的现值

**优点:** 严谨，直接用会计数据
**缺点:** 需要较长期ROE预测

### 4. 相对估值法 - 最实用的工具

基于同业对比：
- **PB倍数对比**：当前PB vs 行业中位数/平均值
- **PE倍数对比**：当前PE vs 行业中位数/平均值
- **股息率对比**：当前股息率 vs 行业平均水平
- **百分位分析**：显示当前估值在行业中的相对位置

## 模型对比与选择指南

### 不同投资者的模型选择

| 投资者类型 | 推荐模型 | 目的 | 关注指标 |
|----------|--------|------|--------|
| **防御型** | PB-ROE + DDM | 安全稳定 | 高股息、低PB、稳定ROE |
| **成长型** | RIV + PB-ROE | 捕捉上升 | 高ROE、城商行、高弹性 |
| **价值型** | 相对估值 + PB-ROE | 寻找底部 | 历史分位、同业对比 |
| **组合型** | 四模型结合 | 综合评估 | 多角度验证 |

### 建议的组合用法

1. **第一步**：用PB-ROE模型快速筛选
   - 计算合理PB区间
   - 识别严重低估品种

2. **第二步**：用DDM模型验证
   - 检验分红理论支撑
   - 评估长期投资价值

3. **第三步**：用RIV模型深度分析
   - 多情景分析（悲观/基础/乐观）
   - 评估超额回报概率

4. **第四步**：用相对估值法确认
   - 与同类银行对比
   - 确认择股逻辑

## 风险提示

### 模型局限性

1. **基于标准假设**
   - 使用的参数（r、g）是行业平均值
   - 个股实际情况可能有差异

2. **数据时效性**
   - 系统使用的是示例数据
   - 实际使用应该接入最新数据源

3. **宏观风险**
   - 利率变化（影响r和g）
   - 经济周期变化（影响ROE）
   - 政策调整（影响行业景气）

### 关键监控指标

| 指标 | 监控重点 | 风险提示 |
|------|--------|--------|
| **净息差** | LPR下行趋势 | 收益率下行压力 |
| **不良率** | 地产/零售压力 | 资本充足率压力 |
| **资本充足率** | 监管要求变化 | 分红/扩张受限 |
| **ROE趋势** | 长期可持续性 | 评估模型参数有效性 |

## FAQ

**Q：为什么宁波银行的PB是1.05而不是破净？**
A：宁波银行ROE高达15.1%，远高于行业平均，合理PB应该>1，所以虽然低于历史高位但不破净。

**Q：如何调整股权成本（r）？**
A：修改`config.py`中的`VALUATION_PARAMS["equity_cost_base"]`，默认10%，可根据市场利率调整为8%-12%之间。

**Q：系统支持实时数据吗？**
A：当前使用的是示例数据。实际使用可修改`data_fetcher.py`中的数据源，接入Tushare、东方财富等实时数据。

**Q：模型什么时候失效？**
A：当发生黑天鹅事件（金融危机、行业剧变）时，历史参数可能失效，需要及时调整阈值。
